## Components

### 1. readmeUpdate.js (메인 스크립트)

- **Purpose:** RSS 피드를 파싱하고 README.md를 재생성
- **Module System:** ES Modules (`"type": "module"` in package.json)
- **동작 순서:**
  1. 프로필 헤더/바디 마크다운 텍스트를 템플릿 문자열로 정의
  2. `rss-parser`로 `https://veritasgarage.tistory.com/rss` 파싱
  3. 최신 글 10개의 제목(title)과 링크(link)를 추출
  4. 마크다운 링크 형식으로 텍스트에 추가 (`#### [title](link)`)
  5. `writeFileSync`로 README.md 덮어쓰기
- **Dependencies:** `rss-parser` (^3.13.0), `node:fs` (built-in)

### 2. GitHub Actions Workflow (.github/workflows/main.yml)

- **Purpose:** README 자동 업데이트 스케줄링 및 커밋/푸시 자동화
- **Trigger:**
  - `schedule`: cron `0 */20 * * *` (20시간마다)
  - `workflow_dispatch`: 수동 실행 지원
- **Runner:** `ubuntu-latest`, Node.js 20
- **Steps:** checkout -> setup-node -> npm ci -> npm start -> git commit/push
- **Git Config:** user `SukbeomH`, email `brent93.dev@gmail.com`

### 3. README.md (생성 결과물)

- **Purpose:** GitHub 프로필 페이지로 표시되는 자동 생성 마크다운
- **구성:**
  - Capsule Render 헤더 배너
  - GitHub Stats + Top Languages 위젯
  - 기술 스택 배지 (Python, JS, TS, Node.js, NestJS, Docker 등)
  - 경력 및 프로젝트 히스토리
  - 블로그 최신글 10개 링크 (자동 업데이트 영역)

## Data Flow

```
1. Trigger (cron 또는 manual)
   └── GitHub Actions 워크플로우 시작

2. Build
   ├── actions/checkout@v3 → 레포지토리 체크아웃
   ├── actions/setup-node@v3 → Node.js 20 설치
   └── npm ci → rss-parser 설치

3. Execute
   ├── npm start → node readmeUpdate.js
   ├── RSS HTTP 요청 → veritasgarage.tistory.com/rss
   ├── XML 파싱 → feed.items[0..9] 추출
   └── writeFileSync → README.md 덮어쓰기

4. Deploy
   ├── git add .
   ├── git commit -m "Update README.md because of Blog updates"
   └── git push → GitHub 프로필 자동 반영
```
